## الوظيفة الأساسية للملف (High-level Purpose)

هذا الملف هو **الأساس النظري والبرمجي لنظام العقد (Nodes System)** في InvokeAI. يُعرّف الكلاسات الأساسية والديكوريترات (Decorators) التي تتطلبها جميع عقد المعالجة. يمكن وصفه بأنه **الدستور البرمجي** الذي تحكم عليه كيفية تعريف العقد وتسجيلها وتنفيذها. يحتوي على أكثر من 800 سطر من الكود الأساسي.

### المدخلات:
- **تعريفات العقد:** جميع ملفات العقد في `invokeai/app/invocations/`
- **الديكوريترات:** `@invocation` و `@invocation_output`

- **كلاسات مسجلة:** جميع أنواع العقد ومخرجاتها مسجلة في `InvocationRegistry`
- **TypeAdapter:** كائن Pydantic للتحقق من الأنواع المركبة

## تفصيل الدوال والأساليب البرمجية (Functions & Methods)

### 1. `Classification` - تصنيف العقد
```
نوع Enum يحدد مستوى استقرار العقد:
- Stable: مستقر - يمكن بناء سير عمل عليه
- Beta: تجريبي - قد يتحطم سير العمل
- Prototype: نموذج أولي - قد يُحذف
- Deprecated: مهمل
- Internal: داخلي
- Special: خاص
```

### 2. `Bottleneck` - تصنيف العقد
```
نوع Enum يحدد ما إذا كان العقدة مقيّدة بالشبكة أو GPU.
```

### 3. `BaseInvocationOutput` - كلاس المخرجات الأساسي
```
الكلاس الأساسي لجميع مخرجات العقد.
```
- **الدالة:** `json_schema_extra()` - إضافة سمات UI إلى مخطط OpenAPI
- **الدالة:** `get_type()` - الحصول على نوع العقدة من الديكوريتر

### 4. `BaseInvocation` - كلاس العقد الأساسي
```
الكلاس المجرد الذي يجب أن تمتد منه جميع العقد.
```

**الخصائص:**
- `id`: معرف فريد لكل نسخة من العقدة (UUID)
- `is_intermediate`: هل هي عقدة وسيطة (لست نهائية)
- `use_cache`: هل يتم استخدام التخزين المؤقت
- `bottleneck`: تصنيف العقدة (GPU/Network)

**الدوال:**
- `invoke(context)`: **الدالة المجردة الرئيسية** - يجب تنفيذها في جميع العقد الفرعية
- `invoke_internal(context, services)`: معالجة داخلية قبل الاستدعاء الفعلي
  - التحقق من الحقول المطلوبة
  - معالجة التخزين المؤقت

### 5. `InvocationRegistry` - سجل العقد
```
فئة ثابتة (Class Registry) لإدارة جميع العقد المسجلة.
```

**الدوال الرئيسية:**
| الدالة | الوصف |
|---|---|
| `register_invocation(invocation)` | تسجيل عقدة جديدة |
| `get_invocation_classes()` | الحصول على جميع العقد مع مراعاة قوائم السماح/المنع |
| `get_invocations_map()` | خريطة الأنواع إلى الكلاسات |
| `get_invocation_for_type(type)` | البحث عن عقدة بالنوع |
| `register_output(output)` | تسجيل مخرج جديد |
| `unregister_pack(node_pack)` | إلغاء تسجيل حزمة عقد |

### 6. `invocation()` - ديكوريتر تسجيل العقد
```
الديكوريتر الرئيسي لتسجيل عقدة جديدة.
```

**المعلمات:**
- `invocation_type`: نوع فريد للعقدة
- `title`: عنوان العرض في UI
- `tags`: وسوم للبحث
- `category`: التصنيف
- `version`: إصدار semver
- `use_cache`: التخزين المؤقت
- `classification`: التصنيف
- `bottleneck`: نوع العقدة

**ما يقوم به الديكوريتر:**
1. التحقق من صحة الحقول
2. إضافة حقل `type` تلقائياً
3. التحقق من دالة `invoke()`
4. تسجيل العقدة في `InvocationRegistry`

### 7. `invocation_output()` - ديكوريتر تسجيل المخرجات
```
الديكوريتر لتسجيل مخرج جديد لعقدة.
```

### 8. `validate_fields()` - التحقق من الحقول
```
دالة للتحقق من:
- عدم تجاوز الحقول المحجوزة لـ Pydantic
- وجود ترميز نوع لكل حقل
- صحة json_schema_extra
- وجود field_kind
```

### 9. `validate_field_default()` - التحقق من القيم الافتراضية
```
دالة للتحقق من صحة القيم الافتراضية باستخدام نموذج Pydantic مؤقت.
```

### 10. `is_optional()` - التحقق من الاختيارية
```
دالة مساعدة للتحقق من أن الترميز اختياري (Optional[X] أو X | None).
```
